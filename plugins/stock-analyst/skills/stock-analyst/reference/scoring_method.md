# 방향성 예측 점수 방법론 (scoring.py)

`scripts/scoring.py`가 산출하는 `forecast.{short,mid}` 필드의 계산 방식을 그대로 문서화한다.
숫자를 서술할 때 근거를 설명해야 하면 이 문서를 참고한다. 검증 에이전트도 "예측이
방법론에 맞게 산출됐는지" 판단할 때 이 문서를 기준으로 삼는다.

**v1 한계**: 이 점수는 과거 데이터로 승률을 검증(백테스트)하지 않았다. 실시간 지표를
정해진 규칙으로 종합한 값일 뿐, 통계적으로 보증된 확률이 아니다. 확정된 미래를
말하지 않는다.

## 1. 신호 컴포넌트 (타임프레임별, -1~+1 정규화)

| 컴포넌트 | 소스 | 정규화 | 가중치 |
|---|---|---|---|
| `trend` | `ma_alignment` | 정배열=+1 / 역배열=-1 / 혼조=0 | 0.30 |
| `macd` | `macd.state` | 골든크로스=+1 / 데드크로스=-1 / 양(+)=+0.5 / 음(-)=-0.5 | 0.25 |
| `rsi` | `rsi14.value` | (값-50)/50 | 0.15 |
| `stochastic` | `stochastic.k,d` 평균 | (평균-50)/50 | 0.10 |
| `bollinger` | `bollinger.pct_b` | (%b-0.5)×2 | 0.10 |
| `range_position` | `range_252.position_pct` | (값-50)/50 | 0.10 |

값이 없는(None) 컴포넌트는 **생략**하고 나머지 가중치로 재정규화한다(0으로 두지 않음 —
"혼조/중립"과 "데이터 없음"은 다른 정보다).

**의도적 선택 — 과매수/과매도를 반전 신호로 쓰지 않는다.** RSI·스토캐스틱·볼린저%b는
값이 높을수록(과매수 방향) 상방 모멘텀이 강하다는 신호로만 쓰고, "그러니 곧 하락"이라는
평균회귀(mean-reversion) 신호로 쓰지 않는다. `reading_indicators.md`의 경고("강한
추세에서는 RSI가 오래 과매수에 머문다")를 스코어링에도 일관 적용한 것이다. 이는 하나의
설계 선택이며, 평균회귀 관점을 쓰는 대안 방법론도 존재한다는 점을 밝혀둔다.

## 2. 타임프레임 → 지평(horizon) 가중

| 타임프레임 | short (1~4주) | mid (1~3개월) |
|---|---|---|
| 일봉(D) | 0.40 | 0.20 |
| 주봉(W) | 0.30 | 0.30 |
| 월봉(M) | 0.20 | 0.35 |
| 연봉(Y) | 0.10 | 0.15 |

타임프레임이 결측(데이터 부족/에러)이면 나머지 타임프레임 가중치로 재정규화하고
`notes`에 남긴다.

`bias_score = round(Σ(타임프레임 signal × 정규화 가중치) × 100)`, 범위 -100~+100.

## 3. 신뢰도(confidence)

`score = 0.45×일치도 + 0.30×가중 데이터커버리지 + 0.25×변동성계수`

- **일치도**: 이 horizon의 가중치 기준 타임프레임 signal 간 가중분산 → `1-표준편차`.
  타임프레임이 1개뿐이면 교차검증 불가로 고정 0.5 페널티. **horizon마다 다르게 나온다**
  (예: 일봉이 상위 프레임과 어긋나면 일봉 비중이 큰 short가 더 크게 감점).
- **가중 데이터커버리지**: 이 horizon이 요구하는 가중치 중 실제 데이터가 있었던 비율.
- **변동성계수**: 항상 **일봉** ATR%(`atr14.pct_of_price`) 기준으로 고정
  `clip(1-(ATR%-5)/15, 0, 1)` — 타임프레임별로 ATR%의 절대 스케일이 달라(월봉 ATR%가
  구조적으로 더 큼) horizon에 맞춰 바꾸면 "장기라서 불확실"이라는 착시가 생기므로
  공통 척도(일봉)로 고정한다.

라벨: `score>=0.65` high / `score>=0.40` mid / 그 외 low.

## 4. 시나리오 확률 (up/sideways/down, 합=100)

```
strength = clip(bias_score/100, -1, 1)
damped   = strength × clip(confidence_score, 0.15, 1.0)   # 신뢰도로 감쇠(과신 방지)
shift    = damped × 33
up       = 33.34 + shift   (최소 1.0)
down     = 33.33 - shift   (최소 1.0)
sideways = 33.33           (최소 1.0)
→ 합이 100이 되도록 정규화 후 반올림, 잔차는 최댓값에 더함
```

신뢰도가 낮으면(`confidence_score`가 0.15에 가까우면) `bias_score`가 커도 `damped`가
0에 가까워져 확률 분포가 중립(33/34/33 근처)에 머문다 — 신호가 약할 때 과신하지 않게
하는 장치다.

## 5. drivers (근거 투명성)

`bias_score`에 대한 각 (타임프레임, 컴포넌트)의 실제 기여도를 계산해 절댓값 기준
상위 6개만 반환한다. 서술 시 "왜 이 점수가 나왔는지"를 설명하는 근거로 쓴다 — 검증
에이전트도 서술의 근거 문장이 `drivers`와 부합하는지 대조한다.

## 6. 검증과의 관계

`scripts/verify.py`는 이 문서의 계약(확률 합=100, 라벨 임계값 0.65/0.40, bias_score
부호와 시나리오 방향 일치)을 결정론적으로 재확인한다. `scripts/test_scoring.py`는
극단 케이스(전 강세/전 약세/결측/단일 타임프레임)에서 이 문서대로 동작하는지 확인한다.
