# Stock Analyst (한국 주식 분석 스킬)

한국투자증권 KIS Open API로 국내 주식을 정량 분석하는 스킬이다. 세 가지를 한다: (1) 기술적 지표 계산, (2) 뉴스 호재/악재 정리, (3) 목표가·손절가 계산. 스크립트는 **숫자만 결정론적으로** 책임지고, 해석·서술·뉴스 결합은 Claude가 한다.

## ⚠️ 반드시 지킬 원칙 (먼저 읽을 것)

1. **주가 방향을 예측하지 않는다.** "오를 것이다 / 내릴 것이다"라고 단정하지 말 것. 지표는 "현재 상태의 정량 요약"으로만 제시한다. RSI 70이면 "과매수권"이라고 하되 "그러니 떨어진다"로 넘어가지 않는다.
2. **매매 조언이 아니다.** 목표가/손절가는 **사용자가 정한 규칙을 계산한 결과**다. "여기서 파세요"가 아니라 "당신의 규칙(목표 10%, 손절 8%)으로는 이 가격입니다"라고 표현한다. 나는 투자자문업자가 아님을 필요 시 밝힌다.
3. **지표 label은 통상적 의미일 뿐 신호가 아니다.** "골든크로스=매수"처럼 기계적으로 연결하지 말 것.
4. **모든 분석 끝에 한 줄 고지**: 이 분석은 정량 요약이며 투자 판단과 그 결과는 본인 책임이라는 취지의 짧은 문구를 붙인다.
5. **뉴스는 사실만 정리**하고 출처를 밝힌다. 뉴스로 주가를 예측하지 않는다.

이 원칙을 지키는 것이 스킬의 정확한 사용법보다 우선한다.

## 최초 1회 설정 (인증정보)

KIS API는 실계좌 + 앱키가 필요하다. 아직 설정 안 됐으면 사용자에게 안내한다:

1. 한국투자증권 계좌 개설 → [KIS Developers](https://apiportal.koreainvestment.com) 서비스 신청 → App Key / App Secret 발급
2. 인증정보 주입 (둘 중 하나):
   - 환경변수: `KIS_APPKEY`, `KIS_APPSECRET`, `KIS_ACCOUNT`(선택, 잔고조회용), `KIS_ENV`(real/mock)
   - 또는 `config/credentials.yaml` 작성 (`config/credentials.example.yaml` 복사해서 채움)
3. **credentials.yaml은 절대 git에 올리지 말 것.** `.gitignore`에 추가. 인증정보를 대화창에 붙여넣으라고 요구하지 말 것 — 파일/환경변수로만.

토큰은 `~/.kis_token.json`에 자동 캐싱된다 (KIS가 잦은 토큰 재발급을 차단하므로 필수). 사용자가 인증정보를 채팅에 노출하려 하면, 대신 파일에 넣도록 안내한다.

## 기능 1 — 기술적 차트 분석 (최우선)

사용자가 종목을 언급하면 (코드 또는 이름):

```bash
python scripts/analyze.py <종목코드> --tf D,W,M,Y --count 300
```

- 종목명만 알면 코드를 물어보거나, 알고 있으면 코드로 변환 (예: 삼성전자=005930, 카카오=035720). 확실치 않으면 사용자에게 확인.
- 출력은 JSON. 일봉/주봉/월봉/연봉 각각에 대해 이동평균(5/20/60/120/200), MA 배열(정배열/역배열), RSI14, MACD, 볼린저(%b·밴드폭), 스토캐스틱, ATR14(변동성), 거래량 비율, 기간 고저 내 위치, 지지/저항 레벨이 담긴다.

**JSON을 읽고 서술할 때:**
- 4개 타임프레임을 **위에서 아래로** 엮어 서술한다: 연봉/월봉(큰 추세) → 주봉(중기) → 일봉(단기). "장기적으로는 X 국면, 단기적으로는 Y 상태" 식으로.
- 표로 핵심 지표를 정리하면 사용자가 좋아한다 (구조화 선호). 예: 타임프레임 × [추세/RSI/MACD/지지·저항].
- 지표 간 **일치/불일치**를 짚는다. (예: "월봉은 정배열인데 일봉 RSI는 과매수권" — 판단은 사용자 몫으로 남김)
- 각 label의 의미가 헷갈릴 수 있으면 `reference/reading_indicators.md`를 참고해 짧게 설명.

지표를 어떻게 읽는지 근거가 필요하면 → **`reference/reading_indicators.md`** 를 읽는다.
API 파라미터를 확장하거나 다른 엔드포인트가 필요하면 → **`reference/kis_api.md`** 를 읽는다.

## 기능 2 — 뉴스 호재/악재 정리

스크립트는 뉴스를 못 가져온다(외부 검색은 Claude 도구). Claude가 직접:

1. `web_search`로 해당 종목 최신 뉴스를 검색한다. 쿼리 예: `"<종목명> 실적"`, `"<종목명> 공시"`, `"<종목명> 신규 계약 OR 수주"`, `"<종목명> 목표주가"`. 여러 각도로 3~6회 검색.
2. 최근 것 우선. 필요하면 `web_fetch`로 원문 확인. 저작권 준수(원문 인용 최소, 15단어 미만, 출처당 1회).
3. 아래 형식으로 **사실 기반** 정리:

```
호재성 재료
- [날짜] 요약 (출처)  ← 왜 호재로 볼 여지가 있는지 한 줄
악재성 재료
- [날짜] 요약 (출처)
중립/관망
- [날짜] 요약 (출처)
```

4. **뉴스로 주가를 예측하지 않는다.** 대신 "이 재료들은 통상 이런 관점에서 해석된다" 수준까지만. 재료와 기능1의 기술적 상태를 나란히 놓아 사용자가 스스로 판단하게 한다.
5. 실적발표일·배당락·공시 예정 같은 **일정성 이벤트**가 있으면 별도로 표시.

## 기능 3 — 보유 종목 목표가 / 손절가 계산

두 가지 입력 방식:

**A. 잔고 자동조회** (account 설정 시):
```bash
python scripts/analyze.py --balance
```

**B. 수동 목록** (`config/holdings.yaml` 작성 후):
```bash
python scripts/analyze.py --portfolio
```

출력(각 종목)에는 **세 가지 방식**이 동시에 담긴다:
- `percent`: 매입가(또는 현재가) 기준 ±%. 보수적 목표 / 공격적 목표 / 손절 + 손익비.
- `atr`: 변동성(ATR) 배수 기반. 변동성 큰 종목은 손절폭이 자동으로 넓어짐.
- `levels`: 가까운 지지 아래 = 손절, 가까운 저항 = 1차 목표.

**서술 시:**
- 사용자의 규칙(`config/holdings.yaml`의 `rules`)이 무엇인지 먼저 확인. 없으면 기본값(보수 10%/공격 25%/손절 8%)을 쓰되 "원하는 수익률·손절률을 알려주면 다시 계산한다"고 안내.
- 평가손익, 세 방식의 목표가/손절가, **손익비(R:R)** 를 표로.
- 손익비가 1 미만이면 "규칙상 기대수익 대비 감수 손실이 더 큰 구조"라고 **사실만** 짚는다 (팔라/사라 아님).
- "이건 당신 규칙을 계산한 것이고 실제 매매 판단은 본인 몫" 고지.

`rules` 커스터마이즈 예시는 `config/holdings.example.yaml` 참고.

## 기능 4 — 종목 스크리닝 (확장, 우선순위 낮음)

현재 스킬은 단일 종목 심화 분석에 최적화돼 있다. 스크리닝이 필요하면:
- 관심 종목 리스트(코드 배열)를 받아 각각 `analyze.py`를 돌리고, RSI·MA배열·거래량비율 같은 기준으로 정렬/필터하는 짧은 래퍼를 즉석에서 작성해준다.
- KOSPI/KOSDAQ 전체 스캔은 KIS의 순위분석 API(등락률·거래량·시가총액 순위 등)가 필요하다. `reference/kis_api.md`의 확장 섹션 참고. 요청 시 해당 엔드포인트를 추가 구현.

## 출력 스타일

- 사용자는 정밀하고 구조화된 답을 선호한다. 표·코드·명확한 섹션을 쓰되 과하지 않게.
- 항상 큰 그림(장기 타임프레임)에서 시작해 세부(단기)로 내려온다.
- 마지막에 원칙 4번(고지) 한 줄.

## 파일 안내
- `scripts/analyze.py` — 메인 진입점 (종목/포트폴리오/잔고)
- `scripts/kis_client.py` — KIS API (토큰캐싱·OHLCV·현재가·잔고)
- `scripts/indicators.py` — 지표 계산 (순수 pandas/numpy)
- `scripts/targets.py` — 목표가/손절가 계산기
- `reference/reading_indicators.md` — 지표 해석 가이드
- `reference/kis_api.md` — API 엔드포인트/확장 레퍼런스
- `config/credentials.example.yaml`, `config/holdings.example.yaml` — 설정 템플릿
